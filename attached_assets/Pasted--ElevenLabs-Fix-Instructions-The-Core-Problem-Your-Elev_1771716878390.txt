# ElevenLabs Fix Instructions

## The Core Problem
Your ElevenLabs tools are calling Supabase edge functions that either:
- Don't exist (contact-form, contact-info → 404)
- Require auth that ElevenLabs can't provide correctly (redirect-to-contact → 401)

## Solution: Two-Part Fix

### Part 1: Update the Supabase Edge Function
In Supabase Dashboard → Edge Functions → `redirect-to-contact`:
- Replace the code with `redirect-to-contact.ts` from this fixes folder
- **CRITICAL**: In the Edge Function settings, set **Enforce JWT Verification** to **OFF**
  (This is the toggle in Supabase Dashboard → Edge Functions → redirect-to-contact → Settings)
  This is why you keep getting 401 — Supabase auto-rejects requests without valid JWT.

### Part 2: Create `contact-info` Edge Function  
In Supabase Dashboard → Edge Functions → Create New → name it `contact-info`:
Paste the code from `contact-info-edge-function.ts` in this folder.
Also set **Enforce JWT Verification** to **OFF**.

### Part 3: Update ElevenLabs Tool Configs

## Why This Works
- Removing auth headers from ElevenLabs config eliminates the secret_id mismatch problem
- Disabling JWT verification on the edge functions lets ElevenLabs call them without auth tokens
- The edge functions are simple passthrough — they accept data and return success
- Security note: These endpoints only accept/return JSON and don't expose sensitive data. 
  If you want auth later, use a simple API key in a custom header, not Supabase JWT.
